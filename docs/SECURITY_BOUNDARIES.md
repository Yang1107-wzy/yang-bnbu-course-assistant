# 安全边界

## 可以做

- 读取当前已登录 MIS 页面的可见 DOM。
- 用户点击`立即启动`或`预约启动`后，从状态页打开实际存在的 ME/FE 详情链接。
- 对当前 BNBU 同源选课页面发送轻量 `HEAD`，只读取标准 HTTP Date 进行校时。
- 对唯一精确匹配目标调用页面已有的 `selectItem`、`selectItemFromWaiting`、`joinWaiting`。
- 短暂包装主页面 `confirm`，只接受当前目标、当前动作的一次确认，随后立即恢复。
- 使用 `.v3` Tampermonkey 本地键同步模式、窗口、校时、worker、FIFO 队列和配置。

## 永远不做

- 不调用 `GM_xmlhttpRequest`，不探测或直调隐藏选课 API，不连接第三方时间服务。
- 不保存登录信息、Cookie、Token、隐藏表单值、姓名或学号。
- 不自动登录、不处理验证码、不绕过访问控制、学校限流或反检测。
- 不执行 `replaceItem`、`dropItem`、`exitWaiting` 或未知函数。
- 不因名称相似、代码部分匹配、班号不同、重复行或函数参数变化而动作。
- 冲刺期详情页刷新不低于 1 秒，普通运行阶段不低于 3 秒。

## 自动停止条件

- 用户点 `Stop` 或按 `Esc`。
- 所有配置目标全部 Selected。
- 预约模式最后一个窗口结束。
- 登录过期。
- 页面函数不存在或抛错。
- 预期确认框未出现、文字不符或出现多次。
- 课程或时间窗口设置非法。

## 剩余风险

浏览器关闭、系统休眠、网络中断或登录过期期间无法执行。恢复后脚本会重新判定窗口，但不会自动登录。正式阶段 HTML、确认文字或页面导航仍可能变化；Test 会显示函数是否 `READY`，未知结构默认不动作。`v1.2.1` 发布前尚未在 2026 年正式选课窗口执行真实提交。使用者应自行确认学校规则和自动化政策，本工具不保证名额或最终选课结果。
