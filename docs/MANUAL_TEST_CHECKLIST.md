# Yang 抢课脚本 v1.2.2 人工验收清单

## 安装与 Test

- [ ] Tampermonkey 只启用一个“Yang 抢课脚本”，版本为 `1.2.2`，许可证显示 `Yang-NCEL-1.0`。
- [ ] 安装内容来自 `dist/yang-bnbu-course-assistant.user.js`，仅匹配两个 BNBU 选课路径。
- [ ] 面板显示 STOPPED/SCHEDULED/RUNNING、北京时间、校时、下一窗口、轮询阶段、Test、立即启动、预约启动、Stop 和设置。
- [ ] `Test` 不触发确认框、不调用选课函数。
- [ ] `Test` 先返回当前页面结果，不等待后台 Worker 全部加载。
- [ ] 默认显示 AI3133 (1001)、COMP4213 (1001)、EBIS3113 (1002)；每个目标的代码、完整名称和班号唯一匹配，页面函数显示 READY。

## 合规确认

- [ ] 从 v1.2.1 升级后首次打开会强制 STOPPED 并显示完整使用声明。
- [ ] 未滚动到底或未勾选时，“确认并继续”保持禁用。
- [ ] 未确认时 Test 和设置可用，立即启动、预约启动、Worker 创建和自动动作均被阻止。
- [ ] 确认后其他标签同步关闭声明，但不会自动开始运行。
- [ ] 完整面板持续显示“仅供学习交流、禁止商业使用、不得用于学校正式选课”的警示。
- [ ] 收起按钮悬停提示和 Worker 迷你状态条仍能看到学习测试用途。
- [ ] 设置区和 Tampermonkey 菜单均可重新打开完整声明。

## 面板布局

- [ ] 标题栏可拖动，点击按钮、输入框和下拉框不会误拖。
- [ ] 右下角手柄可自由调整宽度和高度，面板不会移出屏幕。
- [ ] 点击 `—` 收起为 `Yang` 悬浮按钮；悬浮按钮可拖动，点击 `↗` 恢复原尺寸。
- [ ] 刷新和跳转 ME/FE 后位置、尺寸、收起状态保持。
- [ ] Tampermonkey 菜单可展开或重置面板，收起状态不影响自动监控。

## 北京时间与预约

- [ ] 校时显示 `BNBU SERVER`；失败时显示本机时钟警告而不是伪造成功。
- [ ] 三个默认窗口与学校公布时间完全一致，可分别禁用和编辑。
- [ ] 距下一窗口超过 10 分钟时状态为 SCHEDULED/WAITING，不刷新详情页。
- [ ] 普通阶段显示 NORMAL/3 秒，开放前 30 秒至开放后 2 分钟显示 BURST/1 秒。
- [ ] 当前三门目标只建立 ME-1、ME-2、FE-1 三个 Worker，重复点击启动不会增加标签。
- [ ] Worker 页只有只读迷你状态条；完整按钮只在状态页显示。
- [ ] 手动打开的 ME/FE 详情页显示“Yang 前台优先页”，并遍历本类别所有目标。
- [ ] 已加入轮候后继续监控，已抢到后停止该目标刷新。
- [ ] 到达窗口边界自动从 SCHEDULED 变成 RUNNING；窗口结束自动暂停并等待下一轮。
- [ ] 浏览器标签恢复可见后立即重新判定时间窗口。

## 立即启动与动作

- [ ] 点击`立即启动`后无论当前是否在窗口内都立即 RUNNING，先扫描已打开详情页，再异步打开/复用 worker。
- [ ] 所有配置目标独立入队，前台页候选优先，多门提交动作相隔至少 250 ms。
- [ ] Select 和 Join Waiting List 均不读取学分或轮候人数；合法入口出现后直接执行。
- [ ] 预约启动若当前已在窗口内，也会立即执行 READY 的 Select/Join。
- [ ] 提交后显示 SUBMITTING 并暂停刷新；页面回来后变为 Selected 或 Waiting。
- [ ] 三门全部 Selected 后自动 STOPPED；Waiting 课程继续等待 `selectItemFromWaiting`。

## Stop 与异常

- [ ] `Stop` 或 `Esc` 取消手动运行和所有预约，所有标签不再刷新或提交。
- [ ] 登录过期、函数缺失、参数变化、确认框异常或函数抛错会显示 ERROR 并停止。
- [ ] Clash、重复行、名称/班号不同和未知状态都不动作。
- [ ] 日志不含 URL query、姓名、学号、Cookie 或 Token。
