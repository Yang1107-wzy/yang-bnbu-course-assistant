# 本机合规确认按钮修复设计

日期：2026-07-20

## 问题

首次运行弹窗将“已滚动到底”初始设为 `false`，并且只在 `scroll` 事件中更新。当声明内容能够完整显示、没有实际滚动空间时，不会产生 `scroll` 事件；即使用户已经勾选确认框，“确认并继续”仍保持禁用。

## 已确认方案

取消“必须滚动到底”的交互限制，改为只要用户勾选“我已阅读、理解并同意以上限制”，确认按钮立即可用。

保留：

- 完整的 `Yang-NCEL-1.0` 使用声明；
- 用户主动勾选确认框；
- 合规确认的 GM 存储记录；
- 未确认前保持 `STOPPED` 并阻止自动动作；
- “重新查看免责声明”入口。

不修改：

- 课程配置、Worker、轮询与动作队列；
- Select、Join Waiting List 和确认框执行桥；
- 前台热页优先、1 秒冲刺刷新和 250 ms 动作间隔；
- 许可证文本与持续可见的免责声明；
- GitHub 远程仓库与 Release。

## 实现与验证

1. 先修改测试，使其只勾选复选框、不派发滚动事件，并要求按钮启用。
2. 删除 `scrolledToEnd` 状态和滚动监听；按钮禁用状态只取决于复选框。
3. 运行合规相关测试、全量测试、构建与 dist 安全检查。
4. 重新生成本机 `dist/yang-bnbu-course-assistant.user.js` 和 `release/yang-bnbu-course-assistant.user.js`。
5. 不执行 GitHub 推送。

## 成功标准

- 首次弹窗中勾选复选框后按钮立即可点击。
- 未勾选时按钮仍禁用。
- 点击确认后弹窗关闭并保存有效确认记录。
- 其余 133 项既有行为不回归。
- 构建产物中的 Worker、轮询频率、动作白名单和提交间隔保持原值。

代码验证只能确认本机脚本按设计运行；课程名额、学校服务器响应、登录状态和网络延迟不由脚本控制，因此不承诺最终获得课程。
